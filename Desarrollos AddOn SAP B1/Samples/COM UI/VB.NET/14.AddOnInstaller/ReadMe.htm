Addon Installation Sample

 

The purpose of this sample is to demonstrate how to build an Installer for SBO from scratch. It is not hard to understand or use, but if you simply want to create a simple Installer for your add-on you may find the SBO DE (Development Environment) a better and faster solution. Information about the DE can be found here:

https://www.sdn.sap.com/irj/sdn/developerareas/businessone

 

In order to run this sample you will have to adjust the project.

The Installer project must contain some add-on exe - the add-on to install.

Currently the sample refers to a file called “HelloWorld.exe” but this file is not part of the project, and you will have to add it (or another add-on exe that you want).

It is recommended that you will compile the first sample in the UI – Hello World, and use it as the add-on that will be installed.

 

To achieve that, follow the following steps:

  • Open the HelloWorld project, it should be in “…\SAP Business One SDK\Samples\COM UI\VB6\01.HelloWorld”
  • Make an exe file from the project -
  • Copy the HelloWorld exe file to the Installer project folder.
  • Add HelloWorld.exe as an embedded resource to the project -

 

 

  • Select “Embeded Resource” option in the Build Action property  -

 

  • Now the project is ready to be compiled, compile it and try to run it, the project simulates (by command line parameters) a legal call from SBO for installing.

 

  • Now you have the installer exe file ready for use – “Installer.exe”.

 

If you followed all the steps correctly you have created an add-on file – “HelloWorld.exe” and an installer exe file “Installer.exe”, to understand the whole installation and deployment process you will have to create an ARD file for the add-on.

 

An ARD file is a file created by the utility AddOnRegDataGen (found in "..\SAP Manage\SAP Business One SDK\Tools\AddOnRegDataGen\AddOnRegDataGen.exe")

 

An ARD file contains information about the add-on original exe file, installer file, company name, and some other parameters that you can see here:

 

All this data is coded into the ARD file to make sure that the installer exe is indeed the correct add-on installer file and that the exe of the add-on is indeed the exe expected, and that it was not tampered with or replaced.

 

To make the ARD file fill all the fields correctly and click on “Generate file”, In the Uninstaller command line type “/U”.

 

Now you have everything ready to deploy your add-on in SBO – an Installer exe and ARD file to match it.

 

To Install it simply open the Add-on administration form -

 

And click on “Register Add-On” -

 

Select the exe and ARD files -

 

Click on “Ok” and the add-on will be installed.

 

 

To understand the technical details of installing SBO add-ons read the remarks at the beginning of the Installer sample. It contains very detailed explanation about the Installer.
